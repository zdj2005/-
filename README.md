Library System - 图书借阅系统数据库设计 README

项目概述
本项目是基于 MySQL 开发的图书借阅系统数据库，聚焦图书馆核心业务场景，实现学生与管理员双角色的全流程数字化管理。支持图书查询、借阅申请、审批管理、评论互动等核心功能，确保数据一致性与业务流程闭环。
核心文件说明
1. CREATE TABLEs - 表结构创建脚本
功能：定义系统所需 8 张核心表的结构，包含字段类型、约束条件、外键关联等关键配置。
包含表：
book_type_dict：图书类型字典表（预设 12 类图书类型）
user：用户表（存储管理员、审核员、学生账号信息）
student：学生表（关联用户表，记录学生院系、借阅资格等信息）
book：图书表（存储图书编码、类型、状态等核心信息）
borrow_apply：借阅证申请表（记录学生借阅资格申请记录）
book_borrow：图书借阅表（跟踪借阅、审核、归还全流程）
book_comment：图书评论表（存储学生评分与评论内容）
sys_config：系统全局配置表（存储系统参数配置）
关键特性：
采用 SHA256 加密存储密码，搭配盐值提升安全性
通过外键关联实现表间数据联动，保证引用完整性
新增 CHECK 约束校验图书编码格式，统一时间字段（gmt_create/gmt_modified）规范
2. INSERT DATAs - 数据插入脚本
功能：向核心表插入基础数据与模拟业务数据，满足系统测试与演示需求。
插入数据类型：
基础字典数据：12 类图书类型（计算机 / 编程、文学名著等）
账号数据：2 名管理员、4 名审核员、15 名学生（含完整院系、学号信息）
图书数据：29 本图书（覆盖 8 类类型，含计算机类核心教材）
业务数据：模拟借阅记录（已归还 / 未归还 / 待审核）、借阅证申请记录、图书评论记录（含审核状态）
数据规范：
图书编码遵循「分区_书架号_编号」规则（如 A_01_001）
学生借阅资格默认未通过，需管理员审核激活
借阅期限默认 30 天，自动计算应还日期

核心业务流程
1. 学生视角功能实现：
注册账号 → 申请借阅证 → 审核通过获取借阅资格
多维度查询图书 → 提交借阅申请 → 等待审核
查看借阅记录 → 归还图书 → 发表评论与评分
2. 管理员视角功能实现：
审核学生借阅证申请 → 处理图书借阅 / 归还请求
维护图书信息（新增 / 修改 / 推荐标记）
审核学生评论 → 统计借阅数据（图书排名、学生借阅次数等）
3. 借阅-归还流程功能实现：
学生注册账号-申请借阅证-审核-申请借阅-审核-线下取书-还书-还书审核
